New optional config field: add `brainstorm.panel` under a `brainstorm:` block, defaulting to the five built-in persona ids. ail-brainstorm overhaul: the interview is sized (small or standard), the expert panel roster is WHAT-focused and configurable, panel and adversarial critique run as one merged review Workflow writing structured `review.json` plus `review.md` (replacing `team-review.md` and `adversarial.md`), a findings triage step applies accepted fixes back into the domain files, a one-page `brief.md` synthesis is written last, and a completion contract is tracked in `brainstorm.yaml`. Brainstorm statuses `team-review-done` and `adversarial-done` are replaced by `review-done` and `triaged` (old statuses are still recognized on resume). `render-brainstorm.js` rebuilt as a dashboard with collapsible sections and filterable finding cards. New skill `ail-persona` manages custom review personas under `.ai-lore/personas/`; the generic `brainstorm-panel` agent now receives its perspective spec from the caller. Update `plugin_version` to `0.12.0`.

## Steps

1. If the config does not have a `brainstorm:` block, add one.
2. Under `brainstorm:`, add the following if not already present:
   ```yaml
   panel:
     - product_manager
     - end_user_advocate
     - support_ops
     - business_stakeholder
     - feasibility_scout
   ```
3. Update `plugin_version` to `0.12.0`.
